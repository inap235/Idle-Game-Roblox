## Detailed change notes

1) Duplicate floor bug
- Problem: The Rojo `Baseplate` plus server-created `PlotGround` produced overlapping grassy areas making the world look buggy and visually duplicating objects.
- Fixes:
  - `default.project.json` now sets the `Baseplate` part to `Transparency: 1` and `CanCollide: false` so it won't be visible or collidable in Studio/Play.
  - `src/server/init.server.luau` also attempts to find and hide any `Baseplate` on building a player's plot (defensive runtime fix).
  - The server's `PlotGround` size and position were adjusted so plots are single, solid grounds per player.

2) Progression via seeds (Task 10)
- All non-starter trees include a `seedCost` table in `TreeConfig.Trees` specifying required resources to plant that tree (for example: `RubyTree.seedCost = { Wood = 1000, Apples = 200 }`).
- Client shop displays seed requirements on each card using an orange `🌰` label.
- Server validates that the player has the required resources before allowing a Plant action, deducts the seed resource amounts, and sends a confirmation message when seed resources are consumed.

3) Economics (Task 9)
- `TreeConfig.MarketConfig` expanded with:
  - `taxRate = 0.05` (5% sales tax),
  - `boomChance` and `boomMultiplier` for rare price booms,
  - `minPriceMultiplier` & `maxPriceMultiplier` widened to 0.25 and 2.0 respectively,
  - `recoveryRate` and `dropPerSale` to tune supply/demand.
- Selling resources now computes `grossValue`, calculates `taxAmount`, credits `netValue` to player coins and records tax in the soldResources message.
- Overselling large amounts (server-side threshold: >50 units) triggers a warning notification and depresses price via `dropPrice`.
- Server can trigger random market booms and notifies connected players.
- Client market UI now shows net price after tax and emoji indicators for big swings.

4) Feedback & Consequences (Task 8)
- New or updated messages include emojis and clearer consequences: `treeWithering`, `treeDied`, `deathPenalty`, `witherWarning`, `oversellWarning`, `seedPurchased`, `notEnoughSeeds`, `marketBoom`, `marketCrash`, etc.
- Server sends additional notifications when trees wither or die to remind players of actionable consequences (e.g., revive cost, wasted slot).

5) UI tweaks
- Shop header now: "Seed Shop - Collect resources + coins to plant trees".
- Seed cost label shown on shop card when applicable.
- Market header updated to mention 5% tax and booms.
- Market shows net price after tax (client-side presentation) and highlights large swings with color and emoji.
- Progression header updated to reflect seeds requirement.

## How to test locally (quick)
1. Make sure you have Rojo installed. Build or `serve` the project and open it in Roblox Studio.

```bash
# from repo root
rojo serve
# open Roblox Studio and connect to the local place
```

2. In Studio Play:
- Confirm you only see one grassy ground (the server-created `PlotGround`) and no visible duplicate baseplate.
- Open the in-game menu (M) and inspect the Shop cards — seeds (🌰) should be visible for non-starter trees.
- Try selling resources: note net value after tax in UI and the Notification/Feedback messages.
- Try to plant a high-tier tree without required seed resources — planting should fail with an error message listing missing resources.
- If you manage large sales, verify Notification warns about oversell and prices drop.
